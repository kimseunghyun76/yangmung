# yangmung 가챠 덱 시스템 설계 (Clash Royale 영감) — 제안/보류

> 상태: **설계만**. 구현은 모든 Phase의 **최후순위 재미 요소**로 진행.
> 이미지 생성 명세는 별도 문서 `GACHA_ASSETS_BRIEF.md` (Codex용) 참고.

---

## 0. 한 줄 컨셉

> "학습 한 판을 끝내면 **보석함**이 열리고, 여행 장면 아이콘이 **평이→동→은→금→다이아**로
> 자라나는 **덱**을 모은다. 모아서 업그레이드하면 **심화 팁·특강** 같은 알찬 학습이 해제된다."

핵심은 Clash Royale의 세 가지 쾌감을 학습에 이식하는 것:
1. **수집(Deck)** — 다 모으고 싶다
2. **가챠(설렘)** — 뭐가 나올지 모른다
3. **업그레이드(성장)** — 모을수록 강해진다(=알찬 컨텐츠 해제)

---

## 1. 설계 원칙 (학습 먼저, 게임은 보조)

- **학습이 보상의 유일한 통화.** 보석·조각은 **세션 완수/장면 클리어로만** 획득. 결제로 확률을 사는 구조 금지.
- **Pay-to-win 아님.** 결제는 "확률 뽑기"가 아니라 **컨텐츠 해제(특강 등)** 형태로만(§6, §9).
- **유용한 것이 나온다.** 드롭은 **지금 학습 중인/약한 장면** 위주(§5) — 쓸모없는 중복 더미 방지.
- **학습 흐름을 막지 않는다.** 보석함은 **세션 완료 화면의 마지막 한 박자**(스킵 가능). 별도 그라인딩 강요 X.

---

## 2. 핵심 루프

```
세션 완료(Done) ─▶ [보석함 등장] ─▶ 탭하여 개봉 ─▶ 카드 공개(가챠 연출)
       │                                              │
       └── 장면 클리어/정답률 → 보석함 등급 결정        └── 덱에 적립 → 조각 누적 → 승급 가능 시 알림
```

- 보석함 등급(예: 나무/은/금 상자)은 **성과로 결정**: 별 3개·복구 0 → 더 좋은 상자.
- 개봉은 **선택**(나중에 열기 가능) — 수집/개봉의 설렘을 의식적으로 분리해도 좋음(CR식 "상자 슬롯").

---

## 3. 수집 모델 — 덱 = 여행 장면 카드

- **카드 종류 = 14개 여행 장면**(편의점·식당·전철·호텔·거리·약국·쇼핑·택시·공항·환전·코인로커·택배·라멘·가게).
  → 이미 있는 `scene` 아이콘/색 체계를 그대로 카드화. (학습 지도와 1:1로 맞물림 = 일관성)
- **각 카드는 5단계 외형(티어):**

  | 티어 | 외형 | 의미 |
  |---|---|---|
  | 1 **평이(Plain)** | 현재의 평이한 모노 아이콘 | 막 만난 장면 |
  | 2 **동(Bronze)** | 동 테두리/질감 | 익히는 중 |
  | 3 **은(Silver)** | 은 테두리/광택 | 안정화 |
  | 4 **금(Gold)** | 금 장식 프레임 | 숙달 |
  | 5 **다이아(Diamond)** | 다이아·홀로 효과 | 완성/마스터 |

- 티어는 단순 외형이 아니라 **그 장면의 숙련도/수집도**를 시각화 → "내 일본 여행 실력이 보석처럼 자란다".

---

## 4. 진행 / 업그레이드 (CR식)

- 같은 장면 카드를 **중복 획득 → 조각(shard) 누적 → 임계치 도달 시 다음 티어로 승급**.
- 조각 임계치 예: 동 3 → 은 6 → 금 12 → 다이아 24 (밸런싱 수치, §5).
- **승급 = 학습 마일스톤과 연동(권장):** 단순 중복뿐 아니라
  **그 장면의 미션 완료/복습 안정도**가 충족돼야 상위 티어 승급(게임이 학습을 견인).
  - 예: 금 승급 = 조각 12 **AND** 해당 미션 mastered ≥ 80%.
- 승급 시 **그 장면의 보상 해제**(§6).

---

## 5. 드롭 & 밸런싱

**드롭 후보 가중치 = 학습 단계 매칭(핵심):**
- 지금 **열려 있고 미완료**인 장면 ↑ (바로 쓰는 카드)
- **약점 장면**(적응형 `diagnose.weakScenes`) ↑
- 아직 **잠긴 장면**은 드롭 제외(스포일러·무의미 방지) 또는 낮은 확률의 "예고편" 1장
- 이미 **다이아 완성**한 장면은 ↓ (조각→보석 환원)

**상자별 보상 표(예시, 조정 필요):**
| 상자 | 획득 카드 수 | 상위 티어 확률 | 획득 조건 |
|---|---|---|---|
| 나무 | 1~2 | 동 위주 | 세션 완료 |
| 은 | 2~3 | 은 섞임 | 별 3 또는 장면 클리어 |
| 금 | 3~4 | 금 가능 | 무복구 클리어/연속 학습 |

**안티-프러스트레이션:**
- **천장(pity):** N회 연속 상위 티어 미획득 시 보장.
- **중복 보정:** 완성 카드는 조각 대신 **보석**(통화)으로 자동 환원.
- **확률 공개:** 모든 확률 표를 설정/도감에 **투명 공개**(§9 윤리).

**통화(선택):** 보석(gem) = 중복 환원·소소한 강제 승급용. 학습으로만 획득.

## 6. 언락 보상 설계 (게임 → 학습 환류)

티어 승급이 **학습 컨텐츠를 연다** — 이게 yangmung의 차별점:
- **동/은:** 그 장면의 **심화 팁**(현지 뉘앙스, 흔한 실수), 추가 표현 변형.
- **금:** **미니 특강**(예: "편의점 결제 3종 완전 정복"), 장면 확장 시나리오.
- **다이아:** **마스터 배지** + 그 장면 **고급 회화/발음 클리닉**.
- **유료 학습자 전용 해제:** 금/다이아 보상 중 일부(특강·발음 클리닉·PDF 치트시트)는
  **결제 사용자에게만 해제** — *단, 확률형 결제가 아니라 "해제권" 형태*(§9).

> 즉 결제는 "더 좋은 뽑기"가 아니라 "이미 모은 것의 **알찬 학습 컨텐츠 열람권**". 사행성 회피 + 학습 가치.

## 7. 데이터 모델 (localStorage 확장 — 기존 로직 영향 최소)

신규 키 예: `yangmung:collection:v1`
```ts
interface Collection {
  cards: Record<string /*sceneId*/, { tier: 1|2|3|4|5; shards: number }>;
  gems: number;
  boxes: { grade: 'wood'|'silver'|'gold'; opened: boolean; earnedAt: string }[]; // 미개봉 슬롯
  unlocked: string[]; // 해제된 보상 id (팁/특강)
  seenProb?: boolean;  // 확률 공개 확인
}
```
- 기존 `progress`/`adaptive`는 **읽기만**(드롭 후보·승급 조건 계산). 학습 로직 불변.
- 모두 클라이언트 로컬(현 구조 유지). 결제 해제는 추후 라이선스 플래그로.

## 8. UX / 모션 (개봉 연출)

- **보석함 등장:** 완료 화면 하단에 상자 + "탭하여 열기" 펄스.
- **개봉:** 탭 → 상자 흔들림 → 빛 폭발(ym-burst 확장) → 카드 1장씩 **플립 공개**(티어 글로우 색).
- **신규/승급:** "NEW" 리본 / "동→은 승급!" 카운트업 + 프레임 모핑.
- **도감(Deck) 화면:** 14장 그리드, 티어별 외형, 진행 게이지(조각 n/임계), 미획득은 실루엣.
  - 진입: 홈 또는 완료에서. 복습 탭에 "도감" 추가 가능.
- `prefers-reduced-motion` 존중(즉시 공개 폴백).

---

## 9. 윤리 / 리스크 (반드시 지킬 선)

- **학습 앱 = 사행성 회피.** 가챠의 "설렘"은 살리되:
  - **무과금 획득**(학습으로만), **확률 전면 공개**, **천장 보장**.
  - **현금 → 확률 뽑기 판매 금지.** 결제는 **컨텐츠 해제권**만(§6).
- **수집이 학습을 대체하지 않게.** 보상은 항상 "학습 마일스톤"에 묶음(§4).
- **피로/도파민 루프 주의:** 상자는 세션당 1개, 강제 일일 그라인딩 없음.
- (정직) 가챠는 잘못 설계하면 학습 동기를 외적 보상으로 치환해 **장기적으로 내적 동기를 깎을 수 있음**(과잉정당화 효과). → 보상을 "학습 컨텐츠 해제"로 묶어 외적·내적 동기를 정렬.

## 10. 기대 효과 + 장단점 (요청 정리)

### 기대 효과
- **리텐션↑:** "다음 상자/승급"이 재방문 동기. 짧은 세션 반복과 궁합 좋음.
- **수집 완성 욕구→학습량↑:** 14장 다이아 = 14개 여행 장면 마스터(=실사용 목표와 일치).
- **진척 가시화:** 숙련도가 보석으로 시각화 → 성취감.
- **수익화 경로:** 특강/클리닉 해제권 = 비-사행성 결제 모델.

### 장점
- 학습 목표(장면 마스터)와 **수집 목표가 동일** → 게임이 학습을 견인.
- 기존 자산(scene 아이콘/색·적응형 진단) **재활용** → 추가 학습 콘텐츠 없이도 도입 가능.
- 결제를 해제형으로 두면 **윤리·심사(앱스토어) 리스크 낮음**.

### 단점 / 리스크
- **구현 복잡도 ↑**: 도감·티어·드롭·천장·승급·개봉 연출 + 에셋 70종(또는 프레임 5종).
- **외적 보상 의존 위험**(과잉정당화) — 보상이 학습과 분리되면 역효과.
- **사행성 인상**: 설계 잘못 시 학습 앱 톤 훼손. 확률 공개·무과금 필수.
- **에셋 부담**: 5티어 × 14장 풀세트는 양이 큼(→ §아래 프레임 레이어 방식 권장).
- **밸런싱 지속 비용**: 드롭·임계 수치 튜닝 필요.

**결론(권장):** 도입 가치 있음. 단 **(a) 무과금·확률공개·천장, (b) 보상=학습 컨텐츠 해제,
(c) 에셋은 프레임 레이어 방식**으로 시작하면 리스크 대비 효과가 가장 좋다.

## 11. 단계적 도입 (최후순위)

1. **G1 (MVP):** 완료 화면 보석함(1종) → 개봉 → 그 세션 장면 카드 1~2장 적립 + 도감 화면(그리드·티어 외형·조각 게이지). 통화/천장 없이 수집·승급만.
2. **G2:** 상자 등급(성과 연동) + 드롭 가중치(약점·열린 장면) + 천장/중복 환원(보석).
3. **G3:** 티어 승급 → 학습 보상 해제(심화 팁·미니 특강).
4. **G4:** 결제 해제권(특강/클리닉) + 확률 공개 도감.

---

## 12. 착수 전 결정 필요 항목
1. 카드 단위: **장면 14종**으로 확정? (가나/표현까지 카드화할지 — 권장: 우선 장면만)
2. 승급 조건: 순수 조각 vs **조각 + 학습 마일스톤**(권장 후자)?
3. 에셋: **프레임 레이어(5)** vs 풀세트(70)? (권장 프레임 레이어 — `GACHA_ASSETS_BRIEF.md`)
4. 결제 모델 도입 시점(G4) / 무료 범위.
