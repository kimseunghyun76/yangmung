# yangmung — 빌드 제안서 (토론용 요약)

> 목적: "나라면 이렇게 만들겠다"를 한 장으로. 외부 토론용. 상세 스펙은 PRD.md, 타입은 CONTENT_SCHEMA.md.
> 버전: **v2** · 작성일 2026-05-23 · 상태: 설계 + 검증 하니스 동작, 학습 UI 미구현
> v1→v2 변경: 목표를 "여행 생존"에서 **"정확·빠름·효율적 습득"**으로 끌어올림 + 토론 반영 5보강(§11) — 소리 루프 MVP부터, K↔B↔C 왕복, 마이크로 문법, Recovery 점수화, N5 정확성 체크포인트.

## 1. 목표 (세 축)

- **사용자 목표:** 한국어만 아는 성인이, **일본 여행의 일상 미션**(입국·체크인·주문·전철·길·쇼핑·응급)을 **얼어붙지 않고 완수**. JLPT N5는 덤.
- **습득 목표(v2 격상):** 단지 생존이 아니라 **정확하고·빠르고·효율적인 일본어 습득**. "여행에서 굴러가는 사람"과 "정확히 익히는 사람"을 **동시에**. → 이를 위해 **소리(듣기) 루프·마이크로 문법·N5 정확성 체크**가 MVP부터 필요(§11).
- **앱 목표:** ① 가나를 **자동화 수준**까지 → ② 아이가 듣고 말해온 순서를 **성인용으로 재구성한 입력**으로 → ③ **여행 미션 롤플레이**로 마무리. **텍스트 우선 = 오디오 *자산* 후행이지 *무음*이 아님**(런타임 TTS 듣기 루프는 MVP부터, §11.1).

## 2. 핵심 철학 (한 문장)

> 아이의 발달 순서를 **직역하지 않고**, 알맹이(*고빈도·정서밀착·상황 우선, 문법은 패턴에서 떠오르게*)만 성인 인지에 맞게 압축한다. **입력·상황 우선이라는 경계선을 사수**한다(넘으면 문법-우선 교과서 = 입문자 이탈).

## 3. 구조 — 3트랙 + 2개의 가로축

| 구분 | 내용 |
|------|------|
| **Track K** 문자 자동화 | K1~K11(검증된 가나 11레벨 계승). 인식 → **속도 드릴**. |
| **Track B** 발달식 언어 | B0~B5. B0=소리·반응 워밍업(유아어 아님) → … → B5 동네·거래. 상황 기반. |
| **Track C** 여행 미션 | C1~C7(입국·체크인·주문·전철·길·쇼핑·응급). 분기 시나리오 캡스톤. Irodori Can-do 기반. |
| *N5 Coverage* (가로축) | 트랙 아님. K/B/C에 태그로 붙는 **측정 오버레이**. "덤"이 채워졌는지 집계. |
| *Recovery* (가로축) | 트랙 아님. **못 알아들었을 때 굴러가는 스킬 4종**(다시/천천히/쉽게/보여줘). 모든 미션에 필수. |

## 4. 왜 Duolingo/LingoDeer가 아니라 이걸

- Duolingo=재미는 있는데 문법 약함 / LingoDeer=구조적인데 재미 약함. **둘의 약점이 우리 빈틈.**
- 우리만의 차별점 3가지:
  1. **발달 순서 입력** (고빈도·정서밀착부터, 난이도 절벽 제거)
  2. **Recovery를 설계된 스킬로** (여행자를 "얼어붙는 사람 vs 굴러가는 사람"으로 가르는 지점 — 경쟁 앱 미커버)
  3. **여행 미션 = 최종 산출물** + **한국어 학습자 맞춤**(한국어 대조 tip, 한자 추론 활용)

## 5. 뼈대가 안 흔들리게 하는 법 (이미 동작 중)

과거 실패 모드 = "아이디어는 많은데 콘텐츠 뼈대가 흔들림". 이번엔 **구조적으로** 막는다:

- **단일 소스(SSOT):** CONTENT_SCHEMA.md = 타입·검증 규칙의 유일 기준.
- **원자 + 태깅 그래프:** `KanaItem`/`Phrase`는 참조만, N5/문법은 전역 인덱스.
- **빌드 타임 검증 하니스(V1~V18):** 깨진 참조·표기 위반·중복 도입·미션 품질·복구 규칙을 **빌드 차단**으로. (현재 `npm run lint:content` 0실패/0경고, 테스트 16/16, 빌드 성공.)
- 효과: **콘텐츠가 흔들리면 즉시 빨간불.**

## 6. AI 전략 — 두 군데에만 녹인다

**원칙:** 초보자는 일본어 오류를 판별 못 한다 → **핵심 커리큘럼은 절대 런타임 LLM 생성 금지.** AI는 검증된 뼈대 *위/옆*에서만.

- **(지금) Author-time 엔진:** LLM이 Phrase 후보·오답 보기·미션 분기·복구 경로·한국어 tip·N5 태그를 **생성** → 검증 하니스+사람이 **거른다**. 콘텐츠 600개를 빠르고 안전하게 채우는 가속기.
- **(나중) Runtime 회화 파트너 (킬러 기능):** LLM NPC와 **실제 대화**하되, **우리 Mission 스키마(분기·canDo·레벨·복구)가 가드레일**. 레벨 캡 + 복구 연습 + 한국어 피드백. → **여행 실전의 #1 실패(예상 밖 응답)**를 정적 콘텐츠로 못 덮는 걸 AI가 덮음. 포지셔닝은 "집에서 wifi 실전 연습"(여행지 오프라인 아님).
- **핵심:** 지금 만든 스키마가 곧 **AI를 안전하게 가두는 레일**. 뼈대 작업은 AI 시대에도 안 버려진다.

## 7. 기술 스택

React + Vite + TypeScript (확정). 이유: **위험이 UI가 아니라 콘텐츠 무결성** → TS 타입 + 검증 파이프라인이 핵심. PWA(오프라인), 경량 상태(Zustand/localStorage). 오디오/이미지는 **후행**(검증된 유닛만 kana-master Azure TTS 파이프라인 재사용).

## 8. 로드맵 — 현재 위치와 다음

| 단계 | 상태 |
|------|------|
| 설계(PRD)·스키마(SSOT)·검증 하니스 | ✅ 완료·동작 |
| **MVP 수직 슬라이스** (Phrase 55, Mission 4, KanaItem 8, Unit 8) + **SRS 동작** | ✅ **MVP 검증 통과 (본인·친구·외부 3축) — 외부 배포 준비 완료** |
| 1차 콘텐츠 ~600 (author-time AI 가속) | 이후 (MVP 검증 후) |
| Runtime 회화 파트너 | 그 다음 |
| 프리미엄 오디오/이미지 자산 연결 | 마지막 |

**MVP 슬라이스 범위(60~80 Phrase, A 피드백 반영):**
- **K (유지):** K1~K3 일부 — 읽기 성공감 필수.
- **B (축소, 깎는 축):** **C 미션에 필요한 반응·욕구어만**. B0~B5 전 단계를 넓게 깔지 않음.
- **C (유지·강화, 차별점):** C0 튜토리얼 + C1 편의점 + **C2 식당 주문 한 조각**.
- + Recovery 4종 + **최소 TTS 듣기** + 정확성 팁 1~2개.

> **⚠️ MVP가 검증하지 *않는* 것:** B를 깎았으므로 "성인용 발달식 입력 thesis"는 MVP가 검증 안 함(그건 1차 600에서). **MVP가 검증하는 건 학습 루프 + 여행 실용성.** MVP 통과 ≠ thesis 통과 — 혼동 금지.

**작업 순서 (친구 2라운드 반영, B→C→A):**
1. **C1 편의점 두텁게** (현금/카드/영수증/봉투/포인트/데우기) — "써먹는 느낌"을 가장 빨리.
2. **B0/B1 narrow 보강** — C1/C2가 바로 끌어쓰는 답변 다양화(わかりません·あります·ありません·ほしいです 등). 미션에서 말이 굴러가게.
3. **K2/K3 가나 + 예시단어 소량** — 읽기 진척감만, 많이 X.

> **임계점 분리:** 27 phrase = 흐름 검증 가능. **45~55 = 학습 효과 1차 검증 가능 (사용자 테스트 진입 OK).** 60~80은 MVP 완주 목표지 검증 임계 아님. → 45~55 도달 시 일단 사용자 반응 받고 그 뒤 결정.

> **재우선순위 핵심:** 지금 가장 모르는 건 "콘텐츠 일관성"(해결됨)이 아니라 **"이 학습 방식이 정확·빠르게 통하나"**. 문서가 아니라 **플레이되는 슬라이스 + 소리 루프**로만 검증된다. 그래서 전체 커리큘럼 문서화보다 **얇은 MVP 슬라이스 먼저.** (600은 1차 목표지 MVP 아님.)

## 9. 열린 쟁점 (토론거리)

1. **수직 슬라이스 먼저 vs 커리큘럼 먼저** — 나는 슬라이스 먼저 주장. (반론 환영)
2. **Recovery를 얼마나 강하게** — 미션당 ≥2(성공+부분성공) 규칙이 과한가?
3. **한자 노출 시점** — 가나 우선이지만 언제부터 한자/후리가나?
4. **AI 회화 파트너의 정확성/비용** — 레벨 캡과 가드레일로 충분한가? 과금 모델?
5. **N5를 어디까지** — "덤" 원칙 유지 vs 합격을 정식 목표로 승격?
6. ~~**분량**~~ — *결론: MVP 80~120, 1차 600으로 분리(§8).*
7. **TTS 품질** — 브라우저 Web Speech 일본어 음질이 듣기 훈련에 충분한가, 아니면 핵심 표현만 사전 mp3로?
8. **마이크로 문법 분량** — 10초 팁을 몇 개까지, just-in-time 노출 시점을 무엇으로 트리거?

## 10. 한 줄 결론

> **결정적·검증된 콘텐츠 뼈대 위에**, 발달식 입력 + 여행 미션 + 복구 스킬 + **소리 루프·마이크로 문법·N5 정확성 체크**로, "여행에서 굴러가는 사람"과 "정확히 빨리 익히는 사람"을 **동시에** 만든다. **AI는 콘텐츠 생성 엔진(지금) + 뼈대를 레일 삼은 회화 파트너(나중)** 두 군데. 다음 행동은 **80~120 Phrase + 최소 TTS의 MVP 슬라이스**로 학습 *효과*를 검증하는 것.

## 11. 습득 효율 보강 (v2 — 토론 반영, "정확·빠름·효율" 축)

> 제안서가 *여행 생존*엔 강했지만 *일본어 습득 효율*엔 약했다는 반론을 반영. 큰 방향은 불변, 5개 축만 보강.

**11.1 소리 루프를 MVP부터 (가장 중요)**
"텍스트 우선"은 *오디오 자산* 후행이지 *무음*이 아니다. 초보 실패 1위는 "문장을 몰라서"가 아니라 **"아는 문장인데 못 알아듣기"**. → すみません·お願いします·大丈夫·숫자·금액·역명·봉투/카드/현금은 처음부터 소리로. **MVP부터 런타임 TTS(브라우저 Web Speech) 듣기 루프 필수.** kana-master의 *pre-mp3 우선 → Web Speech 폴백* 패턴 재사용 = 자산 0으로 듣기 확보.

**11.2 K↔B↔C 짧은 왕복 (인터리빙)**
선형 "가나 완성 → 언어 → 미션" 금지(드릴에서 지침). 성인은 "왜 외우나"의 보상이 빨라야 한다.
예: K1(10분) → B0 반응어 → **C0 미니미션**(인사/네/아니요/다시) → K2 → B1 → … *블로킹보다 인터리빙이 효율적.*

**11.3 마이크로 문법 정확도 팁**
"패턴에서 떠오르게"만으론 정확성 부족(は/が, ください/お願いします, です/ます, に/で/へ/を, これ/それ/あれ, 반말/정중). 해결은 강의가 아니라 **10초 대조 팁**.
*가드레일:* 반드시 **한국어 대조 + just-in-time**(그 대비가 처음 중요해지는 순간에만). 안 그러면 문법-우선으로 회귀. (스키마 `tip`/`GrammarPoint` 활용.)

**11.4 Recovery = 보조 바퀴 (점수로 통제)**
강점이지만 과하면 "못 알아들어도 복구하면 되지" 회피 학습 위험. → **점수로 첫 이해를 더 높게**:
첫 시도 이해 **3★** / Recovery 1회 후 성공 **2★** / Fallback action **1★** / 재도전은 Recovery 없이 성공 유도. Recovery는 실패 방지가 아니라 학습 보조 바퀴. (스키마 `recoveryOutcome` 이미 존재.)

**11.5 N5 = 덤(목표) + 정확성 체크포인트(기제)**
"덤" 포지셔닝은 유지(시험대비 트랙 신설 금지). 단 정확성을 위해 **미션 디브리프에 20초 N5식 정확성 체크**를 끼운다: 조사 / 동사 현재·과거·부정 / 형용사 활용 / 수·시간·조수사 / 느린 청해 / 짧은 독해. → **표현은 미션에서 배우고, 정확성은 N5 기준으로 검사.** (N5 Coverage 오버레이가 수동 태그 → 능동 체크로 진화.)

> **한 줄:** *"빠르게 만들기 위한 텍스트 우선" ≠ "빠르게 익히기 위한 소리 기반 훈련." MVP에도 최소 듣기 루프가 들어간다.*
