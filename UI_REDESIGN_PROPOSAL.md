# yangmung UI 재설계 제안 — **Immersive Scene Coach**

> 상태: 제안(구현 전) · 작성 기준 코드: 미션 C0–C13, 적응형 엔진(adaptive.ts), SRS-lite,
> TTS(Web Speech), 모노 아이콘 + 장면 일러스트 에셋, 글래스 시스템(홈 프로토타입 완료).

---

## 0. 문제 정의 (왜 지금 갈아엎나)

기능은 충분히 쌓였다(가나 자동화, 장면 미션, 받아쓰기, 따라말하기, 대화 리캡, 적응형 진단).
그러나 **경험의 얼굴이 아직 "학습 앱 대시보드 + 카드 더미"** 다. 핵심 문제:

1. 카드/패널이 많아 **대시보드처럼** 보인다 — 몰입이 없다.
2. 장면별 연습이 **"목록"** 이지 **"상황 진입"** 이 아니다.
3. 홈의 핵심 CTA(오늘 시작)가 충분히 강하지 않다.
4. 세션이 **텍스트 카드 중심** — 장면·소리·행동이 중심이어야 한다.
5. 아이콘은 좋아졌지만 **감정·몰입을 만드는 이미지/모션**이 부족하다.

**해결 방향:** 앱을 "카드 모음"이 아니라 **편의점·식당·전철 장면 안으로 들어가 짧게 연습하는
여행 시뮬레이터형 학습 경험**으로 재해석한다.

---

## 1. 디자인 방향: **Immersive Scene Coach**

| 축 | 포지셔닝 |
|---|---|
| Duolingo 대비 | 게임이 *먼저*가 아니라 **여행 장면이 먼저**, 게임 요소는 지속용 보조 |
| Pimsleur 대비 | 오디오 중심은 같되 **더 시각적**(장면 비주얼 + 읽기) |
| 가이드북 대비 | 실용적이되 **앱답게 인터랙티브**(듣고·고르고·말하고·쓴다) |
| 일본 감성 | 넣되 **전통풍 장식 과잉 금지**, 절제된 현대 일본 무드 |

**한 줄 컨셉:** "오늘 10분, 편의점 계산을 끝까지 해낸다." — 장면을 클리어하는 코치형 학습.

---

## 2. 경험 원칙 (모든 화면이 따른다)

1. **장면이 주인공.** 화면의 시각적 무게중심은 항상 "지금 어떤 상황인가".
2. **한 번에 하나.** 한 화면 = 하나의 결정/행동. 정보는 하단 패널로 점진 공개.
3. **소리는 보조가 아니라 조작.** 듣기/천천히/전체듣기를 1급 액션으로.
4. **즉각적이고 감정적인 피드백.** 정답·복구·오답·장면 클리어에 모션과 사운드.
5. **개인화가 보이게.** 적응형 진단(약점·수준)을 "오늘 너에게 맞춘" 형태로 노출.
6. **조용한 프리미엄.** 브루탈리즘·네온 금지. 글래스 + 깊이 + 절제.

## 4. 화면별 목표 + 와이어프레임

### 4-1. 홈 — "오늘의 여행 장면"
**목표:** 들어오면 "오늘 뭘, 왜 하는지"가 한눈에. CTA가 가장 강하게.

```
┌─────────────────────────────┐
│ ☾  yangmung        지도 복습 ⚙ │  ← 가벼운 상단 (글래스)
│                             │
│ ╭───────────────────────╮   │
│ │  [편의점 장면 히어로 이미지] │   │  ← 풀와이드 장면 비주얼(오늘의 장면)
│ │                       │   │
│ │  오늘의 장면 · 편의점     │   │
│ │  10분이면 계산을 끝까지   │   │  ← 명확한 커뮤니케이션 목표
│ │  ╭─────────────────╮   │   │
│ │  │   ▶  시작 · 15장   │   │   │  ← 가장 강한 단일 CTA
│ │  ╰─────────────────╯   │   │
│ ╰───────────────────────╯   │
│                             │
│ ◜ 오늘의 코치 ─────────────╮ │  ← 적응형 진단(반투명 패널)
│ │ 약점: し·つ · 듣기 약함    │ │
│ ╰─────────────────────────╯ │
│                             │
│ 가나 안정도   ◐ 42%          │  ← 작게(보조), 링
│ 빠른 연습  [히라][가타][간판][받아쓰기]
└─────────────────────────────┘
```
- 오늘의 장면 1개를 **크게**. 나머지(가나/연습)는 보조로 축소.
- "장면별 연습 목록"은 홈에서 빼고 **지도**로 이동(아래 4-3).

### 4-2. 세션 — 장면 안으로 진입
**목표:** 텍스트 카드가 아니라 **장면 + 소리 + 행동**. 상/중/하 3층.

```
┌─────────────────────────────┐
│ ←            ●●●○○○ 3/6      │  ← 진행 도트(장면 스텝)
│                             │
│   [장면 배경 — 편의점 계산대]    │  ← 상단: 풀스크린 장면(흐릿/오버레이)
│       🧑‍💼 점원                │     NPC/화자 표시
│                             │
│ ╭── 반투명 학습 패널(하단시트) ──╮│  ← 중·하단: 글래스 패널
│ │ 「袋はいりますか？」  ◀)) 🐢   ││  ← 일본어 크게 + 듣기/천천히(핵심)
│ │  봉투 필요하세요?            ││
│ │ ╭───────╮ ╭───────╮       ││
│ │ │ 네     │ │ 괜찮아요 │       ││  ← 선택지(터치) / 말하기 / 행동
│ │ ╰───────╯ ╰───────╯       ││
│ │  🎙 말하기   ✓ 이미 알아요     ││
│ ╰───────────────────────────╯│
└─────────────────────────────┘
   정답→초록 펄스 / 오답→흔들림 / 복구→🛟 슬라이드 / 클리어→축하
```
- 진입 시 **장면 등장 모션** + (옵션)환경음 한 컷.
- 듣기 자동재생은 유지하되, **듣기 버튼을 패널 1급 요소**로.
- 정답/오답/복구/클리어 **모션 명세는 §5-4**.

### 4-3. 지도 — 일본 여행 루트
**목표:** 장면이 **여정처럼 열리는** 경로. 잠김/진행/완료 명확.

```
   공항 ── 전철 ── 편의점 ── 식당 ── 호텔 ── 거리
    ✓       ▶        ◐        🔒      🔒      🔒
   (완료)  (진행)   (진행중)  (잠김)
```
- 세로 스크롤 루트(노드 + 연결선). 노드 썸네일은 지금보다 **"장면"답게**(미니 일러스트).
- 노드 탭 → 해당 장면 인트로 → 세션.

### 4-4. 복습 — 라이브러리
**목표:** "가나 / 표현 / 장면 치트시트 / 약점 재도전" 4구획.
```
[가나]  [표현]  [장면별]  [약점]   ← 세그먼트
─ 선택된 구획 컨텐츠 (글래스 카드 그리드/리스트)
약점 탭 = 적응형 엔진의 weakKana/weakScenes 재도전 진입
```

### 4-5. 설정
학습 모드 / 발음 보조 / 듣기 속도 / 테마(주간·야간·시스템) / 가나 스킵 / 초기화.
하단 시트(글래스)로 통일.

## 5. 디자인 시스템 원칙

### 5-1. 색
- **베이스:** 페이퍼/잉크(현 토큰 유지) + 글래스 반투명 레이어.
- **장면 accent 시스템:** 장소별 색(편의점=블루, 식당=오렌지…)은 **그 장면 안에서만** 주인공.
  홈/지도에선 작은 신호로, 세션에선 배경/포인트로 확장.
- **야간 모드:** 단순 반전이 아니라 "밤의 거리/실내" 무드 — 더 깊은 배경 + 따뜻한 글래스.
- 朱(accent)는 **CTA·정답·중요 신호**에만. 면적 남용 금지.

### 5-2. 타이포 (위계 분리)
- **일본어:** 가장 크고 안정적으로(학습 대상). 본문 22–28px, 자간 여유.
- **한국어 설명:** 깔끔·작게(보조). SF/Apple SD Gothic.
- **읽기 보조(로마자):** 익숙해지면 사라지는 현 정책 유지.
- 라지 타이틀(iOS) 화면 제목.

### 5-3. 아이콘 / 이미지
- **모노 아이콘**(currentColor, 이미 보유) = UI 신호·장면 미니.
- **장면 히어로 일러스트**(`public/scenes/c*-hero.svg`, 현재 미사용) =
  **세션 배경 / 홈 오늘의 장면**으로 재활용. 단, 라이트 하드코딩 → **다크 대응 필요**
  (투명 배경 + 오버레이 그라데이션으로 텍스트 가독 확보, 또는 다크 변형 생성).
- 후속: 장면별 짧은 루프 애니메이션(Lottie/SVG)로 격상(선택).

### 5-4. 모션 (마이크로 애니메이션 명세)
| 이벤트 | 모션 |
|---|---|
| 화면/카드 등장 | 스프링 rise(stagger) — 이미 보유(.ym-rise) |
| 정답 | 초록 펄스 + 체크 팝 + (선택)짧은 사운드 |
| 오답 | 좌우 흔들림(.ym-wrong) + 정답 강조 |
| 복구(🛟) | 옆에서 슬라이드-인 "실패 아님" 배지 |
| 장면 클리어 | 컨페티 + 진행 도트 채움 + 트로피 |
| 장면 전환 | 크로스페이드/슬라이드(상황 이동감) |
| 듣기 재생 | 음파 막대(.ym-wave, 보유) |
- `prefers-reduced-motion` 존중(보유).

### 5-5. 사운드 (학습 UX 중심)
- **듣기 = 1급 조작.** 모든 일본어 줄에 듣기/천천히.
- TTS(Web Speech ja-JP) 기본 + `speakSequence`(대화 전체 듣기, 보유) + `slowListening`.
- **mp3 슬롯 예약**(`audioId`)은 이미 타입에 있음 → 추후 고품질 음성 교체 지점.
- 환경음(편의점 「ピッ」, 전철 안내)은 **후순위·아주 절제**해서 장면 진입 1컷만.

### 5-6. 투명 / 글래스
- 세션은 **장면 배경 + 하단 글래스 학습 패널** 구조.
- 모달·설정·인트로는 **글래스 하단 시트**.
- 글래스 토큰(`--glass-*`)·앰비언트(`.ym-ambient`) 이미 도입(홈 프로토타입).

## 8. 멀티모달 입력 — 기술 현실성(솔직히)

- **듣기:** 지금 가능(Web Speech). ✅
- **말하기(녹음·비교):**
  - 녹음·재생: `MediaRecorder` — 가능.
  - "발음 비교/채점": `SpeechRecognition`(ja-JP)은 **Chrome 계열만 안정**,
    **iOS Safari 지원이 불안정**. → 1차는 **녹음→듣고 셀프 비교**(채점 X)로,
    인식 채점은 가능 환경에서만 옵트인. (현 SpeakCard 철학과 일치)
- **쓰기(가나 획) 인식:**
  - 캔버스 입력 + **획 매칭 휴리스틱** 또는 소형 온디바이스 모델 필요.
  - 1차는 **가나 복습**에 한해 "따라 쓰기(획 가이드)" → 정식 인식은 후순위.
- 결론: **3단계에서 점진 도입**, MVP는 듣기 강화 + 녹음 셀프비교부터.
