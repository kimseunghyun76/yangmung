# 가챠 덱 에셋 생성 명세 (Codex용)

> 짝 문서: 설계/밸런싱/윤리는 `GACHA_DECKS_DESIGN.md`. 이 문서는 **이미지 에셋만** 다룬다.
> 구현은 최후순위지만, 에셋은 미리 생성해 둘 수 있다.

---

## 0. 톤 (반드시 준수)
- 기존 yangmung 톤: **절제된 현대 일본 + Liquid Glass**. 과채도·네온·만화풍 금지.
- 5티어는 "보석/금속 등급"이되 **고급스럽게**(장난감 같지 않게).
- 라이트/다크 양쪽 배경에서 또렷해야 함(카드는 자체 배경 있어도 됨).
- 팔레트 기준: 朱 accent `#b9382e`(라이트)/`#e2655a`(다크), 잉크 `#15140f`/`#f3efe6`.
- 장면 색 체계(편의점=파랑 #2563eb, 식당=주황 #ea580c, 전철=초록 #16a34a … `src/views/scene.ts` 참조) 유지.

---

## 1. 권장 방식 — **프레임 레이어**(에셋 최소화)

70장(14장면×5티어)을 통째로 그리는 대신, **재사용 프레임 5종 + 기존 장면 아이콘 합성**을 권장.

### 산출물 A — 티어 프레임 5종 (장면 공통)
카드 외곽 프레임/배경 트리트먼트. 가운데에 장면 아이콘이 얹힌다.
| name | 티어 | 재질/효과 |
|---|---|---|
| `frame-plain` | 1 평이 | 얇은 모노 라인 테두리, 무채색, 광택 없음 |
| `frame-bronze` | 2 동 | 동(브론즈) 그라데이션 테두리 + 미세 질감 |
| `frame-silver` | 3 은 | 은 광택 테두리 + 약한 하이라이트 |
| `frame-gold` | 4 금 | 금 장식 테두리(코너 오너먼트) + 따뜻한 글로우 |
| `frame-diamond` | 5 다이아 | 다이아/홀로그래픽 테두리 + 무지개 스펙트럼 미광, 별빛 |
- 규격: **카드 비율 3:4** 권장. 권장 1080×1440(@1x 360×480). 투명 PNG 또는 SVG.
- 가운데 **세이프 에어리어**(아이콘 자리) 명시: 중앙 정사각 60% 영역 비움.
- 출력: `public/gacha/frames/<name>.png` (+ @2x).

### 장면 아이콘
- 기존 `public/icons/scene-*.svg`(모노 currentColor) 재사용. 카드에선 **장면 accent 색**으로 칠해 합성.
- 또는 장면별 컬러 일러스트(`public/scenes/c*-hero.svg`)를 축소해 카드 아트로.

> 이 방식이면 신규 에셋 = **프레임 5 + 박스/이펙트(아래)** 뿐. 70장 부담 회피.

---

## 2. 대안 방식 — 풀세트(선택, 비권장 우선순위)
- 14장면 × 5티어 = **70장 카드 아트**를 개별 생성.
- 같은 규격(3:4), 각 카드 = 장면 모티브 + 티어 재질이 통합된 일러스트.
- 톤 일관성/관리 비용이 크므로, 핵심 장면(편의점·식당·전철) 정도만 시범 제작 후 판단 권장.

---

## 3. 산출물 B — 보석함 (가챠 상자)
| name | 설명 |
|---|---|
| `box-wood-closed` | 나무 상자(기본 보상) 닫힘 |
| `box-silver-closed` | 은 상자(별3/클리어) 닫힘 |
| `box-gold-closed` | 금 상자(무복구/연속) 닫힘 |
| `box-open` | 개봉 상태(빛 터지는 공용 베이스, 색은 코드 글로우로) |
- 규격: 정사각 권장 720×720(@1x 240). 투명 PNG/SVG.
- 닫힘 상태는 "탭하여 열기" 펄스용. 개봉 연출 프레임을 여러 장 주면(예: `box-open-1..3`) 시퀀스 가능(선택).
- 출력: `public/gacha/box/<name>.png`.

## 4. 산출물 C — 이펙트/글로우(선택)
- `sparkle` (별빛 입자, 투명), `burst-ray`(빛줄기) — 개봉/승급 연출용. 단색 또는 흰색 베이스(코드에서 틴트).
- 티어 글로우 색은 **코드 토큰**으로 주입(에셋에 하드코딩 금지):
  - plain=무채색, bronze=`#a9743b`, silver=`#9aa3ad`, gold=`#d9a531`, diamond=`#6fd3e6`(홀로).
- 출력: `public/gacha/fx/<name>.png` (또는 SVG).

---

## 5. 기술 규격 요약
- 카드 프레임/박스: 투명 PNG(@1x,@2x) 또는 SVG. 비율 카드 3:4, 박스 1:1.
- 장면 아이콘은 기존 모노 SVG 재사용(currentColor). 프레임은 **장면 색과 독립**(어떤 장면에도 얹힘).
- 색 하드코딩: 프레임의 "금속/보석 재질"만 허용. 글로우/틴트는 코드 주입.
- 파일명 케밥케이스, 위 표의 name 그대로.
- 출력 디렉터리:
  - 프레임 → `public/gacha/frames/`
  - 박스 → `public/gacha/box/`
  - 이펙트 → `public/gacha/fx/`

---

## 6. 코드 연동(생성 후, 설계 문서 §8 참조)
- 카드 렌더 = `<프레임 이미지> + <장면 아이콘(Icon, accent색)>` 합성(절대배치).
- 도감 그리드: 미획득 = 실루엣(프레임만 흐리게 + 자물쇠), 획득 = 현재 티어 프레임.
- 개봉: `box-*-closed` → 탭 → `box-open`(+sparkle/burst) → 카드 플립.
- 티어 글로우/틴트는 `--tier-*` 토큰으로(에셋 무관).
