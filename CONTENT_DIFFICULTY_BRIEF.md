# 콘텐츠 난이도 재정리 명세 (Codex용) — 여행 실전 기준

> 목적: 모든 학습 문장·퀴즈 질문/답변을 **"일본 여행에서 실제로 쓰는" 난이도**로 재정리한다.
> 우리 앱은 **여행 회화 도우미**다. JLPT·시험 대비가 아니다. 난이도도 거기에 맞춘다.

---

## 0. 두 줄 철학 (반드시 준수)
1. **여행 실전 난이도** = 그 표현이 여행에서 얼마나 자주·급하게 필요한가 + 어떤 스킬을 요구하는가.
   문법 복잡도(JLPT 급수)가 아니다. "공항에서 당장 쓰는 한마디"가 "문법적으로 쉬운 문장"보다 우선.
2. **⚠️ 경험치/레벨/계급 ≠ 학습 난이도.** 레벨·계급·가챠 등급(rarity)은 **게임 보상·진척 표시**다.
   이 문서의 난이도(level 1~4)는 **콘텐츠 선택용**이며 경험치와 완전히 별개다. 절대 섞지 말 것.

---

## 1. 학습 난이도 4단계 (앱 모드와 1:1)
| level | 모드 | 누구에게 | 표현 성격 | 보기 표기 |
|---|---|---|---|---|
| **1 입문** | beginner | 처음 접함 | 한두 마디 생존 표현(はい·これください·すみません) | 일본어+한글 |
| **2 기본** | default | 기초 아는 사람 | 짧은 완결 문장, 자주 쓰는 장면 표현 | 일본어+한글 |
| **3 중급** | express | 듣기/회화 어느 정도 | 자연스러운 회화체, 살짝 긴 문장 | 일본어(가나)만 |
| **4 고급** | advanced | 실전 가까움 | 빠른 응답·정중/공손 변형·축약 | 한자 |
> level은 "여행 실전 빈도·상황 압박·스킬 부담"으로 매긴다. 문법명으로 매기지 말 것.

---

## 2. 스킬 축 — 사람마다 강점이 다르다 (재미있게 갈라주기)
학습자는 한 줄(선형)이 아니다. 다음처럼 **스킬별 강·약**이 제각각이다:
- 애니/드라마를 많이 봐서 **듣기·말하기는 되는데 가나를 못 읽는** 사람
- **쓰기**만 약한 사람
- 순차적으로 차근차근 해온 사람
- **완전 처음**인 사람 … 그 외 다양.

그래서 콘텐츠에 **어떤 스킬을 주로 쓰는지/각 스킬 부담이 얼마인지**를 태그해, 앱이 사람 프로필에
맞게 골라줄 수 있어야 한다.

스킬: **읽기(read) · 듣기(listen) · 말하기(speak) · 쓰기(write)**.
- 예) "듣기 강·읽기 약" 프로필 → 듣고 의미 고르기 위주, 가나 읽기는 보조와 함께 천천히.

## 3. 산출물 / 데이터 구조
### (A) 장면 문장 — `src/content/sceneSentences.ts`
현재 `SceneSentence`에 `tier?: 1|2|3`가 있는데, **가챠 카드 등급과 혼동**되므로 의미를 분리·명확화한다:
```ts
export interface SceneSentence {
  id: string;
  kana: string;            // 순수 읽기(한자·문장부호 금지)
  kanji?: string;
  displayKana?: string;
  romaji: string;
  korean: string;
  register: Register;      // receptive(상대 발화) | productive(내 발화)
  speaker: 'clerk' | 'me';
  level: 1 | 2 | 3 | 4;    // ★ 여행 실전 난이도(입문~고급). 기존 tier 대체(경험치 아님)
  skills: ('read' | 'listen' | 'speak' | 'write')[]; // ★ 주로 쓰는 스킬(1개 이상)
  tip?: string;
}
```
- **`tier` → `level`(1~4)** 로 교체(여행 난이도). **`skills`** 추가.
- 빌더(`sentence()`)·`Seed` 튜플·검증기(`sceneSentences.validate.ts`)도 함께 갱신(아래 §6).

### (B) 표현 / 미션 — `phrases.ts`, `missions/*`
- 각 표현·미션 스텝의 선택지가 **그 장면 난이도에 맞는지** 점검. 입문 장면에 고급 표현이 섞이지 않게.
- 미션 대화는 **여행 동선(실제 순서)** 대로, 상대 대사 + 내 답을 자연스럽게(이미 대부분 OK — 점검·보정).

## 4. 분량 / 분포 (핵심)
- **장면당 예시 문장 20개 이상** 유지(현재 40장면 × 20 = 800). 더 풍부하게 가능.
- 한 장면 20개 안에 **난이도 분포**를 둘 것(예: level1 약 6 · level2 약 7 · level3 약 5 · level4 약 2).
  → 같은 장면도 입문~고급이 섞여, 사람 수준에 맞게 골라 쓸 수 있게.
- **전역 중복 0** 유지(장면 간 같은 문장 금지 — 기존 규칙).
- 스킬 태그도 한 장면 안에서 골고루(듣기·말하기·읽기·쓰기).

---

## 5. 여행 팁 (현행 기준 보강)
- `src/content/grammar.ts`의 `category: '문화'|'여행'` 팁을 **현재(2026) 기준**으로 점검·보강.
  (예: 현금/IC, 줄서기, 신발 벗는 곳, 식당 매너 등 — 실전 도움)
- ⚠️ **실시간 정보(환율·날씨)·준비물·입국 QR 등은 이 작업 범위 밖**(아주 나중에 별도 메뉴). 지금은 손대지 말 것.

## 6. 검수 (반드시 통과)
- `npm run lint:sentences` → 0 errors (장면당 20개·전역 중복 0·kana 순수읽기·speaker/register 일치 유지)
  - 검증기에 **level(1~4)·skills(비어있지 않음)** 검사 추가(앱 측에서 함께 반영).
- `npm run build` (tsc) · `npm run lint:content` · `npm run test:content`(17/17) 통과.

## 7. ⚠️ 경험치/계급과의 분리 (다시 강조)
- 가챠 카드 등급(rarity)·트로피·계급·레벨 = **보상/진척(경험치)**. 사용자가 많이 할수록 오른다.
- 본 문서의 **level(학습 난이도)** 는 콘텐츠가 본질적으로 얼마나 어려운가 + 누구에게 맞는가.
- 둘은 **다른 필드·다른 목적**이다. 같은 단어("tier/level")로 섞어 쓰지 말 것.

## 8. (지금 범위 밖 — 메모만)
