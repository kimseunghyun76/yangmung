# 학습 방법 확장 로드맵

> 목표: 학습 퀄리티 향상 — 다양한 출제 방식 + 가나 직접 쓰기.
> 결정(2026-06): 쓰기는 **A→B 단계적**, 신규 학습법 **4종 모두** 채택.
> 원칙: **가나는 항상 무작위 출제, 다음 글자 유추 절대 불가**(이미 `selectScriptKanaCards` 그룹 내 셔플 적용 완료).

기존 카드 종류: `quiz`(읽기/듣기/구분/미션/간판) · `introduce` · `order` · `speak` · `dictation` · `tip` · `discover`.
구현은 **기존 구조 재사용 우선**, 신규 `kind` 는 꼭 필요할 때만.

---

## 1. 듣고 의미 고르기 (청해 강화)  — 우선순위 ★ (저비용)
- **동작:** 표현 음성만 듣고(글자 비표시) 한국어 뜻/상황을 고르기.
- **현황:** phrase `listen:${id}` 듣기 퀴즈가 이미 있음(글자 banner 노출 여부 확인 필요).
  → 글자 숨김 + 한국어 보기 변형(`listenMeaning`)을 추가하거나 기존 듣기 카드의 banner를 가려 강화.
- **데이터:** 추가 콘텐츠 불필요(기존 phrases 재사용).
- **유추 방지:** 보기(오답 distractor)는 같은 장면/품사 군에서 무작위, 정답 위치 셔플(이미 `shuffle` 적용).
- **작업량:** 소. cards.ts 빌더 + 보기 구성만.

## 2. 최소 페어 구분 (발음 변별)  — 우선순위 ★★
- **동작:** 헷갈리는 음을 듣고 둘 중/셋 중 고르기. 예) つ/す, きゃ/か, じ/ち, ー 장음 유무, っ 촉음 유무.
- **현황:** 가나 `confuse` 카드(듣고 글자 고르기)가 부분적으로 존재 → **페어 데이터**로 체계화.
- **데이터(신규):** `src/content/minimalPairs.ts` — `{ id, a, b, focus(설명), audioA/B }` 목록(엄선 20~30쌍).
  - 가나는 `kana` 순수 읽기 규칙 준수. 한자 금지.
- **유추 방지:** 좌우 위치 매번 셔플, 같은 쌍 연속 금지.
- **작업량:** 중(콘텐츠 + 빌더). 콘텐츠는 Codex 위임 가능.

## 3. 한→일 작문 (산출 강화)  — 우선순위 ★★
- **동작:** 한국어 뜻을 보고 일본어 문장을 **가나 타일**(받아쓰기 UI 재사용)로 조립. 정답 비교 채점.
- **현황:** `dictation` 카드(듣고 타일 쓰기)의 UI/채점을 재사용. 프롬프트만 음성→한국어로 교체.
- **구현:** `DictationCard` 에 `promptKind?: 'listen' | 'korean'` 추가(기본 listen) — 최소 변경.
- **데이터:** 짧고 순수 가나인 표현(기존 DICTATION_IDS 풀 + scene 문장 도착 시 확장).
- **유추 방지:** 타일 보기에 distractor 가나 혼합(이미 DICTATION_DISTRACTORS), 타일 순서 셔플.
- **작업량:** 중(카드 필드 + 프롬프트 분기 + 선택 세션).

## 4. 속도전 플래시 (정착·즉답)  — 우선순위 ★
- **동작:** 제한시간(예 카드당 5초) 내 빠르게 푸는 복습 게임 모드. 콤보/점수.
- **구현:** 신규 콘텐츠/카드 불필요 — 기존 quiz 카드를 **세션 래퍼**로 감싸 타이머 UI + 결과 집계.
  - 세션 모드 `flash` 또는 Done/Home 진입점. 타임아웃 = 오답 처리(SRS 반영은 약하게).
- **유추 방지:** 카드 순서 셔플(가나 포함), 같은 답 연속 회피.
- **작업량:** 중(Session 타이머 UI). 콘텐츠 0.

## 5. 가나 직접 쓰기 (A→B 단계적)  — 별도 트랙, 나중
> "유추 절대 불가" → 최종형은 **B(빈 칸 직접 쓰기)**. 먼저 A로 출시 후 B 확장.

### Phase A — 따라쓰기(트레이싱)
- 흐린 가나 가이드 위에 손가락/펜으로 덧쓰기(canvas, pointer events).
- 채점: 기준 획 경로와의 **겹침률/커버리지**(임계값 통과 시 합격). 획순 안내(획별 강조).
- 데이터: 가나별 기준 획 경로 — **KanjiVG**(SVG stroke path) 가나 서브셋 활용 검토.
- 출력: 새 카드 `kind: 'write'`(또는 별도 화면) + 전용 진입점. SRS는 약하게(연습 위주).

### Phase B — 빈 칸 직접 쓰기(시험 모드)
- 가이드 없이 빈 칸에 쓰기 → 기준 획과 **경로/획순 매칭(DTW 등)**으로 정확도 판정.
- 무작위 출제(유추 불가). 오답 시 정답 획순 리플레이.
- 정확도 한계는 학습 임계값으로 보정(완벽 인식 아님 — 합/불 + 피드백).

### 공통 기술 메모(솔직한 한계)
- 완벽한 필기 인식은 어려움 → "정답/획순 피드백 + 임계값 합불"로 학습 가치 확보.
- 전부 **온디바이스/오프라인**(PWA 원칙). 외부 OCR API 미사용.
- iPhone 터치 좌표·고DPI 캔버스·스크롤 충돌 처리 필요.
