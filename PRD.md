# yangmung — 성인용 발달식 일본어 학습 앱 PRD

> 버전: **v0.4** · 상태: 설계(Design) · 구현 전 · 개정일 2026-05-23
> 한 줄 정의: **일본 아이가 듣고 말해온 발달 순서를 성인용으로 재구성한 입력**으로 일본어를 익혀, **일본 여행의 일상 미션을 무리 없이 완수**하고 **N5는 덤으로** 채우게 만드는 앱.

> v0.3 → v0.4 변경 요약 (스키마 직전 정밀화 — **정식 타입은 CONTENT_SCHEMA.md가 SSOT**)
> - **Phrase.japanese → kana(필수, 가나만)/kanji(선택) 분리** + kana-only 린트.
> - **KanaItem 원자화 = 최대 분해(~220)**: 청음+탁음/반탁음+요음+확장가타카나 전부 개별 원자.
> - **K 분량 예산을 KanaItem 수 ↔ 예시단어 Phrase 수로 분리.**
> - **Unit에 newPhraseIds / reviewPhraseIds 분리** (나선형 복습=정상, 예산은 new만 카운트).
> - **MissionStep 구조화**(situationKo/speaker/promptPhraseId) + **register: receptive/productive**(NPC 대사 생산 강요 방지).
> - **SRS 대상 ReviewTarget 유니온**(kana/phrase/grammar/mission).
> - **Track K 받아쓰기 = 텍스트 1차(가나 입력·조립) / 오디오 후행(듣고 입력)** 명확화.
>
> v0.2 → v0.3 변경 요약 (철학 불변, 정밀화만)
> - **B0 재정의**: "유아어 단계" → **"소리·반응 워밍업"** (성인 첫인상 이탈 방지). 유아어는 보너스/모티프 샘플로 강등.
> - **명명 규칙 고정**: 트랙명까지 통일 → **Track K / Track B / Track C**, 레벨 **K1~K11 / B0~B5 / C1~C7**.
> - **KanaItem 모델 추가**: 가나는 Phrase가 아닌 별도 원자(`confusables` 포함). 헷갈림 훈련 가능.
> - **N5 문구 정정**: "공식 목록"이 아니라 **공식=능력 기준 / 어휘·한자·문법 목록=비공식 교재 기반(실무 커버리지)**.
> - **Recovery 4종 세분화**: Repeat / Slow / Simplify / Fallback. 미션 검증 규칙 강화(미션당 ≥2, 성공+부분성공).
> - **분량 예산 신설**: 구간별 목표 수(1차 ≈ 실전 Phrase 600 전후).
> - **Can-do 작성 규칙 신설**: B/C 상황형 템플릿 + K 인식·속도형 변형.
> - **기술 스택 확정**: React + Vite + TypeScript.

## 1. 배경 & 문제의식

kana-master 회고 실패 모드: 콘텐츠 뼈대가 흔들림 / 마지막 단계 난이도 절벽(N5+여행 몰아넣음) / 단순 4지선다 / 이미지·오디오 선작업 부담.

→ yangmung은 **콘텐츠 뼈대를 먼저 동결**, **텍스트만으로 전체 동작**, 검증된 자리에만 오디오/이미지 후행. 뼈대는 **검증 하니스**(§10)로 구조적 강제.

---

## 2. 벤치마킹 (상위=공식 기준, 하위=시장 앱)

### 2.1 상위: 공식 커리큘럼 기준
| 기준 | 채택 이유 |
|------|-----------|
| **Irodori(いろどり)** — 일본국제교류기금 무료 코스 | Can-do · 실생활 상황 기반. "일본 생활을 위한 일본어" = 우리 여행 미션과 동형. **Track C 상위 레퍼런스.** |
| **Marugoto / JF Standard / JF Can-do / CEFR** | Can-do 역량 기술 틀. *주의: 교과서-선형 견인력 → 구조만 차용, 전달은 우리 것.* |
| **JLPT N5** | **측정 기준(커버리지 오버레이)으로만 사용.** ⚠️ **JLPT는 공식 어휘/한자/문법 목록을 공개하지 않는다.** 공식 기준은 "읽기·듣기 능력 범위"로만 정의됨. → **어휘/한자/문법 목록은 JLPTsensei 등 비공식·교재 기반 목록을 "실무 커버리지 기준"으로 사용**(흔히 인용되는 어휘 ~800 / 한자 ~80–103도 비공식 추정치). "공식 N5 목록"이라는 전제를 만들지 말 것. |

### 2.2 하위: 시장 앱 (전달·게임화 참고)
- **Duolingo**: 게임화·습관(연속/XP/리그) 강점, 문법 약함.
- **LingoDeer**: 구조적 문법·문맥 SRS 강점, 재미 약함.
- → **두 약점이 우리 빈틈**: 문맥 SRS + 게임화 + **상황/복구 기반 평가** 결합.

### 2.3 발달 과학 (입력 순서 근거)
NTT 영유아 첫 단어: 의성어·반복어·사회적 루틴어 압도. 마일스톤: 1단어→2단어+첫 질문→조사·문장(2;0–2;6세)→5–6세 핵심 어휘 완성. → **고빈도·정서밀착·상황기반 입력 먼저**. 단 성인은 직역 안 함(§5).

## 3. 목표 & 비목표
**목표:** G1 성인용 재구성 발달식 입력(고빈도·상황 우선) / G2 가나 선행+자동화 / G3 **여행 응용 미션(Track C) 완수 = 최종 산출물** / G4 N5 커버리지 덤 충족(측정) / G5 텍스트만으로 100% 동작 / G6 상황·복구 기반 평가 / G7 검증 하니스로 뼈대 안정.
**비목표:** N4↑ 고급 / 음성인식·발음채점 / 초기 오디오·이미지 제작 / 소셜·리더보드.

## 4. 타깃 사용자
한국어 모국어 성인 입문자(일본어 0). 동기: 여행·K콘텐츠·N5. 전제: 한자는 일부 추론 가능, **가나는 무조건 신규**.

## 5. 학습 철학 — 성인용으로 재구성한 발달식 입력 (불변)

> 아이는 문맹·청각몰입·수년. 성인은 정반대(L1 전이·한자 배경·명시 문법·시간 제약·1일차 문자 필요).
> 발달 순서를 **직역하지 않고**, 알맹이만 — *고빈도·정서밀착·상황기반 입력 먼저, 문법은 패턴에서 떠오르게* — 성인 인지에 맞게 압축.

**사수할 경계선:** 입력/상황 우선. 넘어가 문법-우선 교과서로 회귀하면 입문자 이탈.
**나이 라벨의 위상:** 발달 모티프(0세→초등입학)는 **입력 정렬용 직관 가이드일 뿐**, 실제 목표는 **성인 Can-do**.

## 6. 트랙 구조 (뼈대)

```
Track K  문자 자동화      ── K1~K11 (kana-levels.js 구조 계승). 인식 → 속도 드릴.
Track B  발달식 언어 입력  ── B0~B5 (성인 Can-do 재구성). 상황/복구 기반.
Track C  여행 응용 미션    ── C1~C7 (Irodori Can-do 기반). 점진 해금 캡스톤.
─────────────────────────────────────────────────────────────────
N5 Coverage  ── 트랙 아님. K/B/C를 가로지르는 측정 태그(전역 인덱스).
Recovery     ── 트랙 아님. B/C에 짜여 들어가는 명시 스킬(4종).
```

### 6.1 Track K — 문자 자동화 (K1~K11)
kana-levels.js 11레벨 구조를 **계승**(콘텐츠 신규). 각 레벨 = **인식 → 자동화 드릴** 2국면. 데이터 원자는 `KanaItem`(§9).

| 레벨 | 범위 | 자동화 게이트(예시) |
|------|------|----------------------|
| K1–K3 | 히라가나 청음 46 (あ행~わ행·ん) | 단어 즉독 |
| K4 | 헷갈리는 글자 (あ/お, き/さ, し/つ, ぬ/め, れ/ね…) | 변별 속도 |
| K5 | 히라가나 탁음·반탁음·요음 | 변형 즉독 |
| K6–K8 | 가타카나 청음+변형 (シ/ツ, ソ/ン 변별 포함) | 외래어 즉독 |
| K9 | 가타카나 확장표기 (ファ·ヴ·ティ·チェ…) | 현대어 즉독 |
| K10 | 특수표기 촉음(っ)·장음(ー) | 리듬 정확도 |
| K11 | 종합 복습 + 조사 예외 (は/へ/を 발음) | 문장 즉독 |

> Track K는 시나리오로 못 한다 → **속도 인식·드릴 유지**(상황기반은 B/C 전용).
> **받아쓰기 정의(텍스트 우선):** 1차 = "romaji/한글소리 → 가나 입력" 또는 "가나 조립". 오디오 후행 시 "듣고 입력"으로 확장.

### 6.2 Track B — 발달식 언어 입력 (B0~B5, 성인 Can-do)
실제 콘텐츠·Can-do 문구는 CURRICULUM.md에서 Irodori 대조 후 확정.

| 레벨 | 발달 모티프 | 성인 Can-do (요지) | 대표 콘텐츠(샘플) | 평가 모드 |
|------|-------------|--------------------|--------------------|-----------|
| **B0** | 워밍업 | **일상 고빈도 반응어를 듣고 인지·짧게 반응** | はい/うん/ううん/いや/だめ/もう一回/大丈夫/ありがとう/ごめん/ばいばい | 소리-뜻 매칭, 반응 선택 |
| B1 | 첫 단어 | 욕구·거부를 한 단어로 | これ/ちょうだい/もっと/ない, 몸·음식·가족 | 짝맞추기, 행동 선택 |
| B2 | 두 단어+질문 | 묻고 가리키기 | なに?/どこ?/だれ?, 2어 조합, 색·수 | 빈칸·잇기, Q-A 매칭 |
| B3 | 문장·리액션 | 이해하고 문장으로 답(です/ます 첫 노출) | うん/ううん, 〜がほしい/〜したい, ます형 | 말풍선·리액션 |
| B4 | 사회 언어 | 타인과 예절(인사·부탁·사과·차례) | おはよう/ありがとう/ごめんなさい/かして, 〜てください | 시나리오 분기 |
| B5 | 동네·타인 | 거래·길찾기·숫자·돈·시간 | すみません/これください/いくら/〜はどこですか | 상점·길찾기 미션 |

> **B0 재정의 이유:** まんま·わんわん·いないいないばあ를 첫 화면에 강하게 노출하면 성인 입문자가 "유아어 앱"으로 오해해 이탈. → B0는 **성인이 바로 쓰는 반응어** 중심. **유아어는 보너스/발달 모티프 샘플 카드**로만 둠.

### 6.3 Track C — 여행 응용 미션 (C1~C7, Irodori Can-do 기반)
B 역량이 쌓이는 대로 **점진 해금**. 각 미션 = 실재 Phrase 분기 시나리오 + 별점 평가 + **필수 Recovery 분기**(§6.5, §10).
**초안:** C1 입국심사 · C2 호텔 체크인 · C3 식당 주문 · C4 전철 타기 · C5 길 묻기 · C6 쇼핑·계산 · C7 응급(분실·몸살). (분기 깊이는 CURRICULUM.md.)

### 6.4 N5 Coverage — 가로지르는 측정 오버레이 (트랙 아님)
- 비공식 N5 어휘/문법/한자 목록(§2.1)을 **목록화**하고 K/B/C의 Phrase·GrammarPoint·KanaItem에 **태그 연결**.
- 산출물 N5_COVERAGE.md = "현재 콘텐츠가 실무 N5 목록의 몇 %를 덮는가 / 빠진 항목"을 **자동 집계**.
- 불변식: N5 Coverage는 어떤 트랙/미션의 하위가 아니라 **전역 인덱스**.

### 6.5 Recovery — 명시 스킬 4종 (트랙 아님)
여행자를 "얼어붙는 사람 vs 굴러가는 사람"으로 가르는 핵심. Duolingo/LingoDeer 미커버 빈틈.

| 종류 | 의도 | 대표 표현 | 데이터형 |
|------|------|-----------|----------|
| **Repeat** | 다시 말해 달라 | もう一度お願いします | phrase |
| **Slow** | 천천히 | ゆっくりお願いします | phrase |
| **Simplify** | 쉬운 일본어로 | やさしい日本語でお願いします | phrase |
| **Fallback** | 보여/써 달라·번역앱 | 書いてください / 見せてください / (번역앱 사용) | phrase **또는 action** |

> Fallback은 phrase가 아닌 **비언어 action**(번역앱·제스처)일 수 있어 데이터 모델에서 구분(§9).
> B4부터 도입, **모든 Track C 미션에 Recovery 분기 필수**(규칙은 §10).

## 7. 평가·게임 모드
| 모드 | 적용 | 설명 |
|------|------|------|
| 속도 인식 / 받아쓰기 **드릴** | Track K | 자동화 전용. 시간·정확도. |
| 소리-뜻 매칭 / 반응 선택 | B0 | 가벼운 노출·반응. |
| 짝맞추기 / 빈칸·잇기 | B1–B2 | 인식·조합. |
| 말풍선·리액션 | B3 | 문맥 선택. |
| **상황 행동(action)** | B4–B5, C | "이 상황에서 무엇을 하/말하나". |
| **복구(recovery)** | B4+, C 전 미션 | 못 알아들었을 때 되묻기·우회(4종). |
| 시나리오 분기 미션 | C | 분기·평가·별점. |

**게임화(Duolingo식):** 연속·XP·해금. XP를 발달 진척에 묶어 "다음 나이로 자란다 → 초등 입학 졸업 → 출국" 서사.
**복습/SRS(LingoDeer식):** Phrase 정답률 기반 재출제(경량 Leitner/SM-2). 상황 모드도 SRS 큐 편입.

## 8. Can-do 작성 규칙 (신설 — 콘텐츠가 "문법 설명"으로 새는 것 방지)

### 8.1 B/C 상황형 (기본 템플릿)
> **사용자는 [상황]에서 [상대 발화/정보]를 이해하고, [행동/응답]할 수 있다.**

예: "사용자는 편의점 계산대에서 봉투 필요 여부를 듣고, 필요/불필요를 짧게 답할 수 있다."

규칙: ① 반드시 관찰 가능한 행동으로 끝낼 것 ② 문법명(〜てください 등)을 Can-do에 쓰지 말 것(문법은 패턴 태그로 별도) ③ 한 Can-do = 한 상황.

### 8.2 K 인식·속도형 (문자 전용 변형)
문자엔 "상황"이 없으므로 별도 형식:
> **사용자는 [범위]의 가나를 [기준 속도/정확도]로 읽을 수 있다.**

예: "사용자는 K3 히라가나 청음을 평균 0.8초 내 95% 정확도로 읽을 수 있다."

## 9. 데이터 모델 (개요 — 정식 타입은 CONTENT_SCHEMA.md가 SSOT)

> PRD는 모델의 *의도*만 고정한다. **필드·검증 규칙의 단일 소스(SSOT)는 `CONTENT_SCHEMA.md`.** 두 문서가 충돌하면 CONTENT_SCHEMA가 우선.

핵심 결정(스키마에서 타입으로 확정):
- **원자 2종**: `KanaItem`(Track K, 최대 분해 ~220, `confusables`·`koreanSound`, `examples`는 경량 원시 문자열) / `Phrase`(B/C, **`kana` 필수+가나만 / `kanji` 선택**, `register: receptive|productive`).
- **참조 그래프**: 원자는 포함되지 않고 *참조*만 됨 → 공유 가능.
- **Unit**: `kanaIds`(K) 또는 `newPhraseIds`/`reviewPhraseIds`(B) — 신규/복습 분리.
- **Mission(C1~C7)**: `MissionStep`은 `situationKo`/`speaker`/`promptPhraseId`로 구조화, `Choice`는 phrase형/`actionText`(Fallback) + `recoveryType`/`recoveryOutcome`.
- **전역 인덱스**: `GrammarPoint`, `N5Entry`(source:'unofficial') — Mission 하위 아님.
- **SRS**: `ReviewTarget` 유니온(kana | phrase | grammar | mission).
- 오디오/이미지: `audioId` 비운 채 텍스트로 100% 동작.

## 10. 콘텐츠 검증 하니스 (뼈대 흔들림 방지)

- **타입 강제:** Phrase(kana/korean/register)·KanaItem(char/level/kind) 필수 필드 누락 차단.
- **참조 무결성 린트:** 모든 `kanaIds/phraseIds/unlockAfter/grammarRefs/n5Refs/phraseId`가 실재 id를 가리키는지 검사. 깨진 참조 = 빌드 실패.
- **N5 커버리지 게이트:** 비공식 N5 목록 대비 미커버 항목 빨간불 집계(N5_COVERAGE.md 자동 생성).
- **분량 예산 검사:** §11 구간별 목표 수 미달/초과 경고.
- **Recovery 규칙 (강화):**
  - 각 **Track C 미션당** Recovery 선택지 **최소 2개**(스텝당 아님).
  - 그중 최소 하나는 `recoveryOutcome:'full'`, 하나는 `'partial'` (사소한 미션은 예외 허용).
  - Recovery 사용은 **미션 실패가 아니라 별점만 하락**.
  - 4종(repeat/slow/simplify/fallback) 중 최소 2종 분포 권장.

## 11. 분량 예산 (1차 버전 — 검증 하니스 입력값)

> N5 추정 어휘 800을 억지로 다 채우지 않는다. **1차 = 실전 Phrase ≈ 600 전후 + 문자 자동화 + 여행 미션 완주.**

**Track K (원자 분리 카운트):**
| 항목 | 목표 수 |
|------|---------|
| KanaItem (최대 분해) | ~220 |
| K 예시단어 Phrase | 80~120 |

**Track B/C (신규 Phrase = `newPhraseIds`만 카운트, 복습 제외):**
| 구간 | 신규 Phrase | 누적(K 예시 포함) |
|------|------------|-------------------|
| B0 | 20~30 | 100~150 |
| B1 | 40~60 | 140~210 |
| B2 | 60~80 | 200~290 |
| B3 | 80~100 | 280~390 |
| B4 | 80~100 | 360~490 |
| B5 | 100~140 | 460~630 |
| C1~C7 | 180~260 | 640~890 |

> 1차 목표 = **실전 Phrase ≈ 600 전후 + KanaItem ~220 + 여행 미션 완주.** N5 800 억지 충족 안 함.
> 예산은 `reviewPhraseIds`(나선형 복습)는 카운트하지 않는다 — 재등장은 정상.
