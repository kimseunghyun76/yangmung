# Yangmung Menu Redesign Proposal

## 목적

`docs/new_step.md`의 구조를 Yangmung의 새 학습 흐름으로 채택한다.

새 기준은 다음 순서다.

1. 선행 과정 13개 주제
2. 실전 상황 43개 단원
3. 통합 여행 미션 5개

기존 `C1~C50` 중심의 미션 구조는 바로 삭제하지 않는다. 새 구조가 자리 잡을 때까지 이미지, 도감 아이템, 문장 데이터의 참고 자료로 유지한다.

## 핵심 결정

- 새 커리큘럼의 기준 문서는 `docs/new_step.md`다.
- 기존 `C1~C50`은 더 이상 최상위 학습 순서의 기준으로 쓰지 않는다.
- 메뉴는 "빠른 연습을 많이 펼쳐 놓은 앱"이 아니라 "일본 여행 회화 실전 훈련 앱"처럼 재배치한다.
- 모든 퀴즈는 반드시 새 표현 학습 이후에만 노출한다.
- 도감 보상은 단순 가챠가 아니라 각 실전 단원과 통합 미션의 보상으로 연결한다.

## 현재 메뉴 구조

현재 상단 메뉴는 다음 5개다.

| 현재 메뉴 | 현재 역할 | 문제점 |
|---|---|---|
| 홈 | 진행 현황, 추천, 루트 | 추천 기준이 새 커리큘럼과 직접 연결되지 않는다. |
| 학습 | 빠른 연습, 문자, 어휘, 기능별 연습 | 메뉴가 많고 실제 여행 회화 흐름보다 기능 목록에 가깝다. |
| 미션 | 기존 C 미션 지도 | 새 `new_step.md`의 13+43+5 구조와 맞지 않는다. |
| 복습 | 약점 복습 | 유지해야 하지만 새 단원 단계와 연결이 필요하다. |
| 도감 | 가챠, 아이템 컬렉션 | 기존 C 미션/장소와 묶여 있어 새 커리큘럼과 재매핑이 필요하다. |

## 제안 메뉴 구조

상단 메뉴는 6개로 바꾼다.

| 새 메뉴 | 라우트 제안 | 역할 |
|---|---|---|
| 홈 | `/` | 오늘 해야 할 학습, 다음 단원, 복습, 보상 요약 |
| 준비 | `/prep` | 선행 과정 13개 주제와 문자/단어 기초 |
| 실전 | `/scenes` | 실전 상황 43개 단원 |
| 여행 | `/journey` | 통합 여행 미션 5개 |
| 복습 | `/review` | 오답, 약점, 듣기 실패, 역할극 재도전 |
| 도감 | `/gacha` | 단원별 보상 아이템과 컬렉션 |

모바일에서 6개 탭이 좁으면 상단 탭을 가로 스크롤 가능한 짧은 텍스트 탭으로 만든다. `학습`이라는 큰 메뉴 안에 전부 넣는 방식보다 사용자가 현재 어느 단계에 있는지 바로 알 수 있는 구조가 더 낫다.

## 메뉴별 상세 설계

### 1. 홈

홈은 사용자가 다음에 해야 할 일을 결정해주는 화면으로 바꾼다.

구성:

- 오늘의 다음 학습
- 준비 과정 진행률: `0/13`
- 실전 단원 진행률: `0/43`
- 통합 여행 미션 진행률: `0/5`
- 바로 복습할 오답
- 최근 획득한 도감 아이템
- 권장 학습 순서

추천 기준:

1. 아직 완료하지 않은 필수 준비 과정이 있으면 `준비`를 추천한다.
2. 준비 과정이 충분하면 다음 `실전` 단원을 추천한다.
3. 실전 단원이 묶음 단위로 완료되면 `여행` 통합 미션을 추천한다.
4. 최근 오답이 많으면 `복습`을 함께 제안한다.

기존의 여행 루트 이미지는 홈의 보조 요소로만 둔다. 홈의 중심은 "다음 학습 단위"가 되어야 한다.

### 2. 준비

`docs/new_step.md`의 선행 과정 13개를 중심으로 구성한다.

준비 과정:

| ID | 주제 | 목적 |
|---|---|---|
| P1 | 일본어 발음과 듣기 기초 | 장음, 촉음, 억양, 빠른 직원 말 듣기 |
| P2 | 여행 가타카나 | 역, 호텔, 식당, 쇼핑에서 보는 외래어 |
| P3 | 일본어 문장 기본 순서 | 단어를 문장으로 바꾸는 최소 문법 |
| P4 | 의문사와 질문 만들기 | 먼저 묻는 능력 |
| P5 | 지시어와 위치 표현 | 이것, 저것, 여기, 저기, 방향 |
| P6 | 시간·날짜·수량·금액 | 여행 중 숫자 반응 |
| P7 | 요청·부탁·허락 | 부탁하고 가능한지 묻기 |
| P8 | 선택과 선호 | 자리, 음식, 옵션 고르기 |
| P9 | 긍정·부정·불가능 | 된다는 말과 안 된다는 말 구분 |
| P10 | 이유와 상태 설명 | 문제 상황을 짧게 설명 |
| P11 | 사과와 정중 대응 | 실수했을 때 대응 |
| P12 | 의사소통 복구 | 다시 말해달라고 요청 |
| P13 | 직원에게 자주 듣는 표현 | 서비스 직원의 고정 질문 듣기 |

기존 빠른 연습은 준비 메뉴 안의 보조 훈련으로 재배치한다.

| 기존 기능 | 새 위치 |
|---|---|
| 히라가나 | 준비 > 문자 |
| 가타카나 | 준비 > 문자 |
| 가나 쓰기 | 준비 > 문자 훈련 |
| 가나 말하기 | 준비 > 발음 훈련 |
| 발음 구분 | 준비 > 듣기 기초 |
| 받아쓰기 | 준비 > 듣기 기초 |
| 어휘 | 준비 > 단어 |
| 동사 형태 | 준비 > 문장 만들기 |
| 기본 인사 | 준비 > 기초 표현 |
| 간판 읽기 | 준비 > 여행 표지 |

빠른 연습은 없어지는 것이 아니라 "준비를 보완하는 연습 도구"가 된다.

### 3. 실전

실전 메뉴는 43개 상황 단원을 노출한다.

그룹:

| 그룹 | 단원 | 주제 |
|---|---|---|
| S1 | 1~6과 | 생존 회화와 대화 시작 |
| S2 | 7~13과 | 공항 도착과 기본 이동 |
| S3 | 14~23과 | 숙박·식사·일상생활 |
| S4 | 24~33과 | 장거리 이동·쇼핑·관광 |
| S5 | 34~40과 | 문제 해결과 안전 |
| S6 | 41~43과 | 출국 |

실전 단원 카드는 다음 정보를 보여준다.

- 단원명
- 실제 상황 이미지
- 필요한 준비 과정
- 핵심 표현 수
- 직원이 하는 말 수
- 진행률
- 완료 보상 아이템
- 오답 복습 여부

실전 단원은 기존 미션 지도처럼 랜덤 해금하지 않는다. 여행 동선과 난이도에 맞춰 순서대로 열리는 것이 기본이다. 단, 설정의 접근 제한 해제 옵션이 켜져 있으면 모든 단원을 열어 디버깅할 수 있게 한다.

### 4. 여행

여행 메뉴는 통합 미션 5개를 담당한다.

| ID | 통합 미션 | 포함 흐름 |
|---|---|---|
| J1 | 일본 도착일 | 입국, 교통카드, 전철, 호텔 체크인 |
| J2 | 관광과 식사 | 길 찾기, 입장권, 식당, 계산 |
| J3 | 지방도시 여행 | 신칸센, 버스, 료칸, 온천 |
| J4 | 문제 발생일 | 분실, 병원, 예약 변경, 교통 장애 |
| J5 | 출국일 | 공항 이동, 체크인, 보안검색, 탑승 |

통합 미션은 여러 실전 단원을 연결한 장기 세션이다.

잠금 기준:

- J1은 S1~S14 핵심 단원을 완료해야 열린다.
- J2는 식당·관광 관련 단원을 완료해야 열린다.
- J3는 장거리 이동·숙박 관련 단원을 완료해야 열린다.
- J4는 문제 해결 단원을 완료해야 열린다.
- J5는 출국 관련 단원을 완료해야 열린다.

통합 미션은 보상도 일반 단원보다 크게 준다. 도감에서는 통합 미션 전용 아이템 또는 배경 배지를 제공할 수 있다.

### 5. 복습

복습은 단순 오답 노트가 아니라 "실제 여행 대응력"을 회복하는 메뉴로 바꾼다.

복습 묶음:

- 틀린 객관식 다시 풀기
- 잘 못 들은 직원 질문 다시 듣기
- 한자 표기와 히라가나 표기 재확인
- 분기 대화에서 실패한 선택 다시 보기
- 내가 말해야 하는 표현 다시 말하기
- 통합 미션 중 실패한 장면만 재도전

복습은 홈과 세션 완료 화면에서도 계속 추천한다.

### 6. 도감

도감은 새 실전 단원과 연결한다.

기존 문제:

- 기존 아이템이 `C1~C50` 또는 `place` 기준으로 묶여 있다.
- 같은 장소가 반복될 때 아이템이 중복되거나 맞지 않을 수 있다.
- 일부 아이템은 실제 일본 여행에서 보기 어렵다.

새 기준:

- 도감 아이템은 `unitId` 기준으로 연결한다.
- `P` 준비 과정은 작은 기초 보상만 준다.
- `S` 실전 단원은 실제 장소/상황별 아이템을 준다.
- `J` 통합 미션은 고급 보상 또는 세트 보상을 준다.

아이템 예:

| 단원 | 아이템 방향 |
|---|---|
| S18 식당 입장과 대기 | 대기표, 메뉴판, 테이블 번호표 |
| S19 메뉴 확인과 기본 주문 | 규동, 라멘, 정식, 장어덮밥 |
| S21 음식점 유형별 주문 | 라멘, 회전초밥, 야키니쿠, 이자카야 |
| S23 편의점과 셀프 계산대 | 삼각김밥, 편의점 치킨, 도시락, 디저트 |
| S31 신사와 사찰 | 5엔 동전, 오미쿠지, 에마, 오마모리 |

도감 이미지 생성은 단원 목록이 확정된 뒤 다시 진행한다. 기존 `docs/gacha-item-redesign.md`는 참고용으로 두되, 새 단원 ID 기준으로 다시 정리해야 한다.

## 단원 내부 학습 흐름

각 준비 과정, 실전 단원, 통합 미션은 같은 학습 골격을 가진다.

| 단계 | 이름 | 내용 | 기존 컴포넌트 대응 |
|---|---|---|---|
| 1 | 상황 미리보기 | 장면 이미지, 목표, 오늘 배울 표현 | `Preview` 계열 |
| 2 | 핵심 표현 | 단어, 히라가나, 한자 표기, 뜻 | `IntroduceCard` |
| 3 | 직원이 하는 말 | 현지 직원 질문 듣기 | 듣기 퀴즈 |
| 4 | 기본 대화 | 가장 예측 가능한 대화 | `Mission.steps` |
| 5 | 분기 대화 | 다른 선택지와 문제 상황 | `choices`, `feedback` 확장 |
| 6 | 자유 역할극 | 사용자가 직접 말하며 진행 | 신규 카드 필요 |
| 7 | 듣기 평가 | 실제 속도에 가까운 듣기 확인 | 받아쓰기/듣기 퀴즈 확장 |
| 8 | 완료/보상 | 복습 추천, 도감 보상 | `Done` 화면 개편 |

퀴즈 잠금 규칙:

- 같은 단원 안에서 핵심 표현을 한 번도 보지 않은 문제는 절대 퀴즈로 나오면 안 된다.
- 직원이 하는 말도 먼저 듣기 소개를 거친 뒤 평가로 나온다.
- 복습에서도 사용자가 학습한 적 없는 표현은 섞지 않는다.

## 데이터 모델 제안

새로운 최상위 단위는 `unit`으로 통일한다.

```ts
type UnitKind = 'prep' | 'scene' | 'journey';

type UnitId = `P${number}` | `S${number}` | `J${number}`;

interface LearningUnit {
  id: UnitId;
  kind: UnitKind;
  title: string;
  subtitle: string;
  stageGroup?: string;
  requiredUnitIds: UnitId[];
  image: string;
  blocks: LessonBlock[];
  rewardIds: string[];
}

type LessonBlockKind =
  | 'preview'
  | 'coreExpression'
  | 'staffLine'
  | 'basicDialogue'
  | 'branchDialogue'
  | 'roleplay'
  | 'listeningAssessment';
```

기존 `Mission` 데이터는 한 번에 버리지 않는다. 우선 `LearningUnit`으로 감싸는 어댑터를 만들어 새 화면에서 읽을 수 있게 한다.

## 화면 구성 방향

### 홈

- 큰 대표 이미지 1개
- 다음 학습 카드 1개
- 진행률 3개: 준비, 실전, 여행
- 복습 CTA
- 최근 보상

### 준비

- 상단: 현재 권장 준비 주제
- 중단: P1~P13 카드
- 하단: 문자/어휘/받아쓰기/말하기 등 빠른 연습 도구

### 실전

- 6개 그룹 탭
- 각 그룹 안에 단원 카드
- 단원 카드는 1열 2행 또는 2열 그리드로 크게 노출
- 이미지는 기존 quick-practice 수준의 고품질 장면을 사용

### 여행

- 5개 통합 미션을 큰 루트 카드로 노출
- 각 미션은 여러 장면이 이어지는 여정 이미지 사용
- 실패한 장면만 재도전 가능

### 세션 완료

`다음엔 무엇을 할까요?`는 새 기준으로 바꾼다.

우선순위:

1. 틀린 문제가 있으면 `오답만 다시 잡기`
2. 아직 덜 본 핵심 표현이 있으면 `다시 한번 학습`
3. 다음 단원이 열렸으면 `다음 실전 단원`
4. 통합 미션 조건을 만족하면 `여행 미션 시작`
5. 보상이 있으면 `도감 확인`

## 콘텐츠 제작 순서

1. `P1~P13`, `S1~S43`, `J1~J5` ID를 확정한다.
2. 각 단원에 필요한 핵심 표현, 직원 질문, 기본 대화, 분기 대화를 표로 만든다.
3. 기존 C 미션 문장 중 재사용 가능한 문장을 새 단원으로 매핑한다.
4. 새 단원 대표 이미지를 생성한다.
5. 단원별 도감 아이템을 다시 지정한다.
6. 음성 생성 대상 문장을 카운트한다.
7. P1~P3, S1~S3, J1만 먼저 파일럿 구현한다.
8. 화면 구조와 학습 잠금 규칙을 검증한 뒤 전체 단원으로 확장한다.

## 구현 단계

### Phase 1. 문서와 ID 확정

- `docs/new_step.md`를 기준 문서로 확정한다.
- `P/S/J` ID 목록을 별도 데이터 파일로 만든다.
- 기존 `new-c1-c50-travel-scenes.md`는 참고 자료로만 둔다.

### Phase 2. 네비게이션 개편

- 상단 메뉴를 `홈 / 준비 / 실전 / 여행 / 복습 / 도감`으로 바꾼다.
- 기존 `/practice`, `/map` 접근은 호환 처리한다.
- 설정의 접근 제한 해제 옵션은 유지한다.

### Phase 3. 단원 데이터 어댑터

- 기존 `Mission`을 새 `LearningUnit`으로 감싸는 어댑터를 만든다.
- 새 표현 학습 이력과 퀴즈 잠금 규칙을 `unitId + phraseId` 기준으로 저장한다.

### Phase 4. 파일럿 콘텐츠

- P1: 일본어 발음과 듣기 기초
- P2: 여행 가타카나
- S1: 직원에게 말 걸기
- S2: 알아듣지 못했을 때
- J1: 일본 도착일

위 5개만 먼저 구현해서 흐름을 검증한다.

### Phase 5. 도감과 보상 재연결

- 도감 아이템을 `unitId` 기준으로 재설계한다.
- 기존 가챠 이미지 중 재사용 가능한 것은 보존한다.
- 새 아이템 이미지는 전체 단원 목록이 확정된 뒤 생성한다.

### Phase 6. 음성/역할극 확장

- 직원이 하는 말, 기본 대화, 분기 대화, 듣기 평가 문장을 음성 생성 대상에 넣는다.
- 자유 역할극은 처음에는 선택지 기반으로 시작하고, 이후 음성 입력 기반으로 확장한다.

## 남은 결정 사항

- 상단 6개 탭을 그대로 둘지, 모바일에서는 일부를 하단/더보기로 보낼지 결정해야 한다.
- 기존 `C1~C50`을 사용자에게 완전히 숨길 시점을 정해야 한다.
- 도감의 무료 가챠를 계속 둘지, 학습 보상 중심으로 축소할지 결정해야 한다.
- 자유 역할극을 텍스트 선택형으로 먼저 갈지, 음성 입력형까지 바로 갈지 결정해야 한다.
