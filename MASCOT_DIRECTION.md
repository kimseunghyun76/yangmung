# yangmung 마스코트 방향서 — yang & mung

> 상태: **확정용 기준 문서**. 구현 AI는 이 문서를 단일 기준으로 반영한다.
> 핵심 한 줄: **"yang과 mung이 학습을 같이 해준다"** 는 감정선으로 앱의 개성을 만든다.
> 단, **학습이 항상 주인공**이고 마스코트는 보조다(절제).

## 1. 캐릭터 정의

### 🐱 yang (고양이) — "차분한 설명·정확성 코치"
- 역할: **설명 / 힌트 / 읽는 법 / 정확성**. 새 표현·팁·문법·가나에서 등장.
- 성격: 똑똑하고 차분, 군더더기 없음. 정확하지만 다정.
- 말투(기본 **해요체**): 담백한 설명조. "이건 이렇게 읽어요", "여기선 이 표현이 자연스러워요".

### 🐶 mung (시바) — "따뜻한 응원·회복 코치"
- 역할: **응원 / 완료 / 복구 / 재도전**. 정답·오답·복구·장면 클리어에서 등장.
- 성격: 활기차고 따뜻, 실패를 격려로 바꿈.
- 말투(기본 **해요체 + 가벼운 활기**): "좋아요, 한 걸음 더!", "막혀도 괜찮아요 — 이렇게 넘어가면 돼요".

### 👯 duo (둘 함께)
- **홈 / 완료 / 특별 보상**에서만. 일상 학습 화면에서 둘이 동시 등장 금지(산만).

> 말투 존댓말/반말: 기본은 **둘 다 해요체**(앱 보이스와 일관). mung을 더 캐주얼(반말)하게 갈지는 §10 결정 항목.

## 3. 말투 / 카피 원칙
- **1줄, 짧게**(권장 ≤ 22자). 학습 대상(일본어)을 절대 가리지 않게.
- 한국어. 이모지 남용 금지(0~1개).
- yang=설명조, mung=격려조. **정답을 미리 흘리지 않기**(힌트는 "읽는 법/접근"까지만).
- 같은 멘트 연속 금지 → 상황별 **카피 풀에서 회전**(§7).
- 평가절하·부정 금지("틀렸어" X → "이 상황엔 이게 더 자연스러워요" O).

---

## 4. 금지선 (반드시 지킴)
1. **학습 흐름 방해 금지** — 탭/진행을 막거나 모달로 가리지 않는다. 자동 진행 타이밍을 늦추지 않는다.
2. **정답 누설 금지** — 퀴즈 전 정답/선택을 마스코트가 말하지 않는다.
3. **빈도 제한** — 세션 내 마스코트 말풍선은 **연속 등장 금지**(쿨다운: 최소 2~3카드 간격, 또는 정답/오답 등 이벤트성에만).
4. **모션 절제** — 등장은 부드럽게(ym-rise), `prefers-reduced-motion`이면 정적. 통통 튀는 과한 루프 금지.
5. **톤 유지** — 유아틱/네온 금지(앱의 절제된 글래스 톤 안에서). 얼굴 크기 과대 금지.
6. **접근성** — 장식용 얼굴은 `alt=""`/`aria-hidden`, 의미 있는 멘트는 텍스트로도 읽힘.
7. **학습 대체 금지** — 마스코트가 채점/정오를 대신하지 않는다(피드백 로직은 그대로, 마스코트는 감정선만).

## 5. UI 적용 원칙 (구현 가이드)
- **공용 컴포넌트 2개**(신규, `src/views/mascot.tsx` 권장):
  - `<MascotBubble who="yang"|"mung" text size? tone? />` — 얼굴(webp) + 글래스 말풍선 1줄. 등장 시 `ym-rise`.
  - `<MascotFace who size />` — 얼굴만(빈 상태/로딩/인라인).
- **톤 일치**: 말풍선은 글래스(`--glass-bg-strong` + `--glass-border`), 텍스트 위계는 본문보다 작게(13~14px), 朱는 강조에만.
- **빈도 제어**: 세션 마스코트는 App에서 가벼운 게이트(예: 직전 등장 카드 인덱스 기억 → 간격 보장, 또는 정답/오답/복구 이벤트에만).
- **데이터/로직 불변**: 마스코트는 **표시 레이어만**. 진척·SRS·적응형·TTS 로직 건드리지 않음.
- 성능: webp만 사용(이미 PNG 원본 제거됨), `width/height` 지정·`loading="lazy"`.

## 6. 감정 피드백 정리(마스코트와 함께)
- 정답: 초록 펄스(기존) + mung 가끔 1줄.
- 오답: 흔들림(기존) + mung "이 상황엔 이게 자연스러워요" + 자연스러운 답(기존).
- 복구: "전략 — 실패 아님" 배지(기존) + mung 격려.
- 장면 클리어/완료: 여권 스탬프(기존) + duo 축하.
- → **기존 피드백 UI는 유지**, 마스코트는 그 위에 "사람 같은 한마디"만 얹는다.

## 7. 카피 뱅크 (예시 — 회전용, 구현 시 풀로)
- **인트로(yang):** "오늘은 {장소}예요. 같이 가봐요." / "{장소}에서 쓰는 말, 천천히 익혀요."
- **정답(mung):** "좋아요, 그거예요!" / "자연스러웠어요 👍" / "한 걸음 더!"
- **오답(mung):** "괜찮아요, 이 상황엔 이게 더 자연스러워요." / "거의 다 왔어요."
- **복구(mung):** "막혀도 이렇게 넘어가면 돼요." / "실패 아니에요 — 좋은 전략!"
- **팁/문법(yang):** "여기선 이 한 끗이 포인트예요." / "이렇게 읽으면 돼요."
- **새 표현(yang):** "이 표현 같이 봐요." / "소리부터 익혀요."
- **완료(duo):** "오늘도 한 장면 해냈어요!" / "여권에 도장 하나 더 🛂"
- **빈 상태(yang):** "아직 비어 있어요 — 한 판 하면 채워져요."
> 각 풀은 3~6개, 직전 멘트 반복 금지.

## 8. 에셋 현황 & 추가(선택, Codex)
- **MVP:** 현재 얼굴 2종 + duo 로고로 충분(감정은 텍스트가 담당).
- **표정 변형(선택, 차별화↑):** yang(설명/생각/안심), mung(응원/기쁨/토닥) 각 2~3종 webp.
  - 규격: 정사각 투명 webp, 얼굴 크롭, 동일 화풍, @1x 256/@2x 512. 출력 `public/mascots/<who>-<expr>.webp`.
  - 이 변형은 **MASCOT_ASSETS 추가 명세**로 분리 생성 권장(필요 시 별도 문서화).

## 9. 구현 단계
1. **M1:** `mascot.tsx`(MascotBubble/MascotFace) + 카피 뱅크(상수) + 빈도 게이트.
2. **M2:** 위치별 삽입 — 인트로(yang 말풍선) → 세션 정답/오답/복구(mung) → 팁/새표현(yang) → 완료(duo) → 빈상태/로딩(얼굴).
3. **M3(선택):** 표정 variants 도입.
- 각 단계 후: build·lint·test(17/17)·라이트/다크 육안.

## 10. 결정 필요 항목
1. 말투: 둘 다 **해요체**(기본) vs **mung만 반말 캐주얼**?
2. 세션 마스코트 빈도: **이벤트성(정답·오답·복구)만** vs 간격 기반 등장?
3. 표정 variants(§8) **지금 만들지** vs MVP는 얼굴 1종으로?
4. 인트로 말풍선 카피: 장소명 자동 삽입("오늘은 편의점이야") OK?
