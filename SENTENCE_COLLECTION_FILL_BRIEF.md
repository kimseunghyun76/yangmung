# 문장 도감 채우기 명세 (Codex용) — 장면당 정확히 20개·전역 중복 0

> 목적: 각 장면(C1~C40)의 **문장 도감을 정확히 20개**로, **장면 전용 내용만**, **다른 장면과 절대 중복 없이** 채운다.
> 현재 공통 베이스(COMMON)는 제거됨. 기존 미션 대사와 장면별 보충 문장을 합쳐 장면마다 20개를 만든다.

---

## 0. 한 줄 요약
`src/content/sceneSentences.ts`의 **`SPECIFIC`/`SUPPLEMENTAL` 레코드**를 수정해, **C1~C40 각 장면이 정확히 20개의
SceneSentence**를 갖게 한다. **40×20 = 800개 문장의 `kana`가 전역적으로 모두 고유**해야 한다(어떤 두 장면도
같은 문장을 공유하지 않음).

---

## 1. 수정 대상 / 구조
- **파일:** `src/content/sceneSentences.ts`
- **수정:** `const SPECIFIC: Record<SentenceScene, Seed[]>`와 `const SUPPLEMENTAL: Partial<Record<SentenceScene, Seed[]>>`.
- **건드리지 말 것:** `Seed` 타입, `sentence()` 빌더, `SCENE_SENTENCES` export, COMMON 재추가.
- `buildScene(scene)`은 `SPECIFIC + missionSeeds + SUPPLEMENTAL` 순서로 읽고, 전역 중복을 건너뛴 뒤 20개에서 멈춘다.

### Seed 형식 (튜플)
```ts
type Seed = [kana: string, kanji: string, korean: string, speaker: 'clerk' | 'me', tier?: 1|2|3, tip?: string];
```
- `id`·`romaji`·`register`는 빌더가 자동 생성하므로 **쓰지 않는다**(Seed에는 위 6개만).
- `speaker: 'clerk'` = 상대(점원·직원 등) 발화 / `'me'` = 사용자 발화.

---

## 2. 핵심 규칙 (검수 — `npm run lint:sentences` 0 errors 필수)
1. **개수:** C1~C40 결과 = **정확히 20개**.
2. **전역 중복 0:** **모든 장면을 통틀어 `kana`가 중복되면 안 됨**(한 문장은 한 장면에만). ← 최우선
   - 같은 의미라도 장면이 다르면 표현을 달리해 고유하게(예: 감사 표현을 장면마다 다른 문장으로).
3. **장면 전용:** 그 장면(장소·상황, §3 표)에서 실제로 쓰는 문장만. 다른 장면 문맥의 문장 금지.
4. **`kana`(순수 읽기):** 히라가나/가타카나 + 장음 `ー`만. **한자·문장부호·공백 금지.**
   (검수 정규식: `^[぀-ゟ゠-ヿー]+$`) 예) `りょうがえおねがいします`
5. **`kanji`:** 자연 표기(한자·문장부호 OK). kana와 같으면 빌더가 자동 생략. 예) `両替お願いします`
6. **`korean`:** 자연스러운 한국어 뜻(필수).
7. **`speaker`/register 일치:** clerk는 receptive, me는 productive로 자동 매핑 → 의미상 맞게.
   (검수: clerk가 productive거나 me가 receptive면 에러 — 즉 점원 대사는 clerk, 내 말은 me)
8. **정중체(です/ます) 기본**, N5~N4 여행 수준, 원어민 자연스러움.
9. **구성 권장(강제 아님):** 한 장면 20개 안에 점원(clerk) 7~10 + 나(me) 10~13, 난이도 tier 1~3 섞기.

### 기존 항목 처리
- 각 장면에 이미 있는 좋은 문장은 **유지**, 부족분만 추가해 20개로.
- **장면 간 중복되는 문장**(여러 장면에 같은 kana)이 있으면 **한 장면에만 남기고 나머지는 그 장면 고유
  문장으로 교체.** (현재 중복은 빌드에서 가려지지만, 정확히 20개를 위해 원천 제거)
